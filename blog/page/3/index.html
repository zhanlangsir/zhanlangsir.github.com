从BlogEngine把数据转到WordPress

Jun 23rd, 2011

从BlogEngine转到WordPress，可以按照以下步骤转换。

1.导出BlogML.xml文件

2.更改BlogML.xml里面的文件路径（例如image.axd?…之类的）

3.更改BlogML.xml里的分类的GUID，更改为名称即可（记得全局匹配）

4.用BlogML插件(也可以直接插件页面搜索一下就能找到)导入BlogML（上传到/wp-admin/import/然后导入->blogml）

5.如果导入有错误，可以用Bulk Delete删掉。

6.如果导入超时，可以在BlogML那个php头文件的require_once(‘XPath.class.php’);之后加入如下代码。

ini_set(“memory_limit”,”146M”); ini_set(“max_execution_time”, “300″); ini_set(“max_input_time”, “300″);

OK，反复操作吧，直到成功。

Fedora15下安装mp3解码器

May 24th, 2011

由于版权问题，fedora是没有安装mp3解码器的，官方源里同样没有，所以对于新手来说，要听mp3还得折腾一翻，所以这里就介绍一下安装mp3解码器的方法，让新手少走点弯路。

首先就是添加有mp3解码器的下载源-fusion
fusion free
fusion nonfree
先下载上面两个rpm文件，下载完安装后就会添加上了fusion源了。

然后打开终端,输入

su 密码
yum --nogpg install gstreamer-ffmpeg gstream-plugin-bad  gstream-plugin-ugly

然后再打开mp3文件看看，是不是已经可以了～  

获取 GPG 密钥失败：[Errno 14]

May 24th, 2011

每个 fusion源发布的稳定 RPM 软件包都配有一个 GPG 签名。默认情况下，yum 和图形更新工具验证这些签名并拒绝安装任何没有签名或者签名损坏的软件包。您总是应该在安装软件包之前验证其签名。这些签名可确保您要安装的软件包出自fusion仓库，且没有被提供该软件包的网页或者镜像更换（无意的或者恶意的）。

而我们安装fusion源的时候，默认是没有添加gpg密钥的，所以安装软件的时候会出现像
获取 GPG 密钥失败：[Errno 14] Could not open/read file:///etc/pki/rpm-gpg/RPM-GPG-KEY-rpmfusion-free-fedora-15-x86_64
这种错误，那么怎么解决呢？答案当然是导入密钥了～

硬盘安装ubuntu11.04

May 20th, 2011

今天想升级到11.04，刻盘后发现刻坏了，无法启动，真悲剧，只能硬盘安装了~

  1. 下载软件
    操作系统ubuntu：ubuntu
    系统引导软件EasyBCD：easybcd
  2. 设置NeoGrub启动项
    1）把ubuntu的iso文件放到C盘根目录，并将iso中的casper目录下的vmlinuz和 initrd.lz解压到C盘根目录下
    2）安装EasyBCD
    3）启动EasyBCD
    4）选择 Add New Entry -> NeoGrub -> Install -> Configure 将如下代码粘贴到自动打开的记事本中 
     title Install Ubuntu 10.10
 root (hd0,0)
 kernel (hd0,0)/vmlinuz boot=casper iso-scan/filename=/ubuntu-11.04-desktop-i386.iso ro quiet splash locale=zh_CN.UTF-8
 initrd (hd0,0)/initrd.lz

    注：记事本代码中的ubuntu-11.04-desktop-i386.iso要和下载的ubuntu文件名字一致

  3. 硬盘安装ubuntu
    1）重启电脑选择NeoGrub Bootloader启动项进入ubuntu live cd桌面
    2）打开终端，或者alt+F2,输入如下命令
    sudo umount -l /isodevice
    3）点击桌面上的安装ubuntu一步步按照提示选择安装
  4. 调整默认启动项为WIN7
    1）终端执行：
    sudo gedit /boot/grub/grub.cfg
    2）找到default 0把0改为4就可以了（4为WIN7的启动项顺序号）
  5. 删除NeoGrub启动项
    1）进入WIN7桌面启动EasyBCD，选择 Add New Entry -> NeoGrub -> remove 删除NeoGrub启动项
    2）删除C盘根目录下的vmlinuz和 initrd.lz文件

那么ubuntu的硬盘安装就完成了~是不是很简单呢~

在model的一对多映射中,发现导航属性无法正常获取，通过下断点查看,存的时候确实有存进去,而且对应的数据库中也已经出现导航属性对应的model的数据,但是在读取该导航属性的时候，却发现结果为null,即没有正确映射到model上。后来再看了一遍教程的说明,发现少了virtual属性,以致
