.NET Reflector 7.1.0.143 破解

Jun 28th, 2011

什么是.NET Reflector?

一般会去查找它的破解版的同学肯定知道它是干什么用的~不过这里还是简单说明一下吧~

.NET Reflector是一个类浏览器和反编译器，可以分析程序集并向你展示它的所有秘密。.NET 框架向全世界引入了可用来分析任何基于 .NET 的代码（无论它是单个类还是完整的程序集）的反射概念。反射还可以用来检索有关特定程序集中包含的各种类、方法和属性的信息。使用 .NET Reflector，你可以浏览程序集的类和方法，可以分析由这些类和方法生成的 Microsoft 中间语言 (MSIL)，并且可以反编译这些类和方法并查看 C# 或 Visual Basic .NET 中的等价类和方法。

在 .NET Reflector 内部，有各种可用来进一步分析该程序集的工具。要查看构成某个方法的 MSIL，请单击该方法并从菜单中选择 Disassembler。

.NET 框架提供了许多执行类似操作的不同方法。例如，如果你需要从 XML 中读取一组数据，则存在多种使用 XmlDocument、XPathNavigator 或 XmlReader 完成该工作的不同方法。通过使用 .NET Reflector， 你可以查看 Microsoft 在编写数据集的 ReadXml 方法时使用了什么，或者查看他们在从配置文件读取数据时做了哪些工作。.NET Reflector 还是一个了解以下最佳实施策略的优秀方法：创建诸如 HttpHandlers 或配置处理程序之类的对象，因为你可以了解到 Microsoft 工作组实际上是如何在框架中生成这些对象的。

官网：http://www.reflector.net/

从BlogEngine把数据转到WordPress

Jun 23rd, 2011

从BlogEngine转到WordPress，可以按照以下步骤转换。

1.导出BlogML.xml文件

2.更改BlogML.xml里面的文件路径（例如image.axd?…之类的）

3.更改BlogML.xml里的分类的GUID，更改为名称即可（记得全局匹配）

4.用BlogML插件(也可以直接插件页面搜索一下就能找到)导入BlogML（上传到/wp-admin/import/然后导入->blogml）

5.如果导入有错误，可以用Bulk Delete删掉。

6.如果导入超时，可以在BlogML那个php头文件的require_once(‘XPath.class.php’);之后加入如下代码。

ini_set(“memory_limit”,”146M”); ini_set(“max_execution_time”, “300″); ini_set(“max_input_time”, “300″);

OK，反复操作吧，直到成功。

Rhythmbox经常崩溃或者自动关闭

May 29th, 2011

linux下毕竟没有像foobar2K这样的音乐播放器，又不想用wine,对比了下linux下的播放器，最终还是选择了rhythmbox，banshee虽然看起来不错，但小狼试用了一下，发现音质确实悲剧，有同学可能会说，rhythmbox和banshee用的都是gstream输出，音质是一样的，但实际不一样～小狼也不知道为什么，banshee确实会出现破声,声音不清晰等等问题。但是用了一段时间的rhythmbox后，发现只要先切换到rhythmbox然后按alt+tab切换，rhythmbox就会崩溃掉，同样在左侧菜单栏随便点点某些选项也会自动关闭，真是太坑爹了。。。

翻了一下google,发现是配置文件惹的祸，只要删掉对应配置文件就可以了，不用怕，下次导入音乐的时候，配置文件会自动重新创建的～

rhythmbox的配置文件在以下几个地方，即在用户主目录下按ctrl+h,显示隐藏文件就可以看见了。

~/.gconf/apps/rhythmbox
~/.cache/rhythmbox
~/.local/share/rhythmbox
~/.gnome2/accels/rhythmbox

然后就把他们都删掉就可以了！

Fedora15下安装mp3解码器

May 24th, 2011

由于版权问题，fedora是没有安装mp3解码器的，官方源里同样没有，所以对于新手来说，要听mp3还得折腾一翻，所以这里就介绍一下安装mp3解码器的方法，让新手少走点弯路。

首先就是添加有mp3解码器的下载源-fusion

http://download1.rpmfusion.org/f … e-stable.noarch.rpm

http://download1.rpmfusion.org/n … e-stable.noarch.rpm

先下载上面两个rpm文件，下载完安装后就会添加上了fusion源了。

然后打开终端,输入

su

密码

yum –nogpg install gstreamer-ffmpeg gstream-plugin-bad  gstream-plugin-ugly

然后再打开mp3文件看看，是不是已经可以了～
