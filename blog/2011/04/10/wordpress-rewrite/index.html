WordPress的Rewrite问题

Apr 10th, 2011

昨天配置完php的环境,今天当然是要动手本地整一下wordpress啦,所以老是睡懒觉的小狼今天早早就起了床,着手把aitilife整站转移到本机上进行调试,一切有条有理的进行。但是…在本地导入数据库,修改网站url,转移主题和插件，什么都完成后,事情的高潮来了… 本地的wordpress表面,后台看起来一切正常,但是一点击任何文章的链接或者分类目录,标签之类的就出事了,提示说无法找到这个网页。小狼就纳闷了,这是肿么回事呢？数据库方面没有任何问题啊,难道环境搭建错了?也不应该啊,昨天已经测试好的了。 (注：点击查看环境搭建相关文章) 百思不得其解的时候，突然很嗨地发现IIS还没有装上urlrewrite，而站点的url都是经过优化的,所以会出现这样的问题,发现问题之后,马上就要消灭他。过程如下： 1.下载iis的url rewrite module，32位下第一个,64位下第二个 Download the x86 version of the URL Rewrite module Download the x64 version of the URL Rewrite module 2.下完后,开始->运行->服务,打开服务列表,找到[more] Windows Process Activation Service和World Wide Web Publishing Service 右键点击它们，停止掉，然后开始安装url rewrite module 3.安装完重启后,打开iis管理器,找到你的网站,这时候中间哪会多出一个url rewrite出来，双击他,进入url rewrite页面,点右上的Add Rule(s)… 然后选择Blank rule,确定,然后会出现规则填写页面: name一栏随便写，最好写点有意义的,例如Main Rule。 然后Pattern一栏也随便写,因为这里写的是网站根目录的，而我们需要的是根目录下的wordpress博客的 ,其他情况就别乱写了 拉到下面的Rewrite URL，也可以随便写，写完就点右边的应用.关掉IIS管理器. 注意:文章后面有web.config文件下载,如果是下载了web.config的话,直接复制到你的wordpress博客根目录下就可以了,那第四步就可以直接跳过！ 4.进入你的网站根目录,看看是不是多了一个web.config文件,把他复制到你的wordpress博客的根目录下,然后添加下面的规则到 system.webServer节点，现在先不用管这些是什么意思,有空的话小狼会写一篇文章详细介绍rewrite规则的编写的! 最后的结果应该像这样的: 好了,再打开你的本地wordpress博客看看,现在是不是全部正常了。 web.config下载地址:http://www.gokuai.com/f/T06l8q8B1b686X51
